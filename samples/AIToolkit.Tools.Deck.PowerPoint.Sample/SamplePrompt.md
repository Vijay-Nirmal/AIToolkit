# Comprehensive PowerPoint Agent Prompt

Use this prompt with the sample agent when you want one end-user request that produces a polished deck while exercising the supported PowerPoint feature set.

```text
Create a new PowerPoint presentation at presentations/strategy-offsite.pptx in the local workspace.

Create a polished executive presentation for a two-day strategy offsite for a leadership team. It should feel business-focused, presentation-ready, and visually intentional rather than like a technical demo, while still showcasing the supported PowerPoint authoring surface naturally. Aim for a premium keynote-style look with strong hierarchy, cinematic composition, bold section openers, and a curated design language.

Use the seeded hero image, logo, and badge as branded assets. The cover must visibly feature all three and feel like a real opening slide. Keep recurring branded structure across the deck with layered compositions, full-bleed or treated imagery where appropriate, intentional whitespace, and repeated motifs such as tinted panels, section dividers, metric cards, and accent lines.

Bias the deck toward visual storytelling over dense paragraphs. Most slides should have a focal visual such as a hero image treatment, data visualization, icon cluster, timeline, comparison panel, layered callout, or structured card layout. Use text sparingly so the slides feel built for live presentation, not document reading.

Include a mix of content that naturally exercises the supported feature set:
- reusable branded slide designs
- rich backgrounds and visual structure
- speaker notes on every visible slide
- slide transitions and at least one staged animation sequence that improves the storytelling
- one grouped visual composition that behaves like a coordinated element
- text, subtitle, caption, image, icon, shape, and line-based visual elements
- disc, dash, check, and numbered lists used in places where each style makes sense
- at least one data table and one chart with clear labels and business meaning
- one richer callout or emphasis block that uses more detailed text formatting than a plain text box
- one hidden appendix slide for backup material
- several slides with high visual density built from multiple coordinated elements instead of a single text box and title
- tasteful use of shadows, transparency, cropping, layering, color blocking, and emphasis treatments where supported
- motion that feels deliberate: staggered reveals, progressive list builds, chart or proof-point sequencing, and transitions that support pacing rather than distract

Make the deck cover these business moments:
- a strong cover slide for the offsite theme
- an agenda or roadmap slide
- a proof-points slide built around metrics, risks, or operating results
- a decision-framing slide that makes leadership choices explicit
- a hidden appendix with supporting detail
- at least one visually dramatic section-break slide between major topics
- at least one slide that uses a dashboard-like arrangement of metrics, mini-charts, or status indicators

Use supported visual options intentionally: image treatment, chart presentation, list progression, animation timing, transition behavior, emphasis callouts, layout reuse, accessibility text, and polished speaker notes. Use multiple animation and effect styles when they suit the story, including entrance emphasis, sequencing, wipes, fades, motion paths or object movement if supported, and visual emphasis that guides attention across the slide.

Vary composition from slide to slide while keeping the design system consistent. Mix full-bleed visuals, split layouts, layered cards, highlighted quote or callout slides, and data-forward layouts. Style charts and tables to match the deck rather than looking like default Office inserts.

Keep everything fully on-canvas, readable, and cleanly composed with generous spacing, clear alignment, accessible contrast, and speaker notes that explain the narrative and intended animation pacing for each visible slide.

Keep complex slides well structured. Cards, sidebars, dashboards, and comparison panels should feel intentionally composed, with enough internal spacing that labels, values, lists, icons, and rules do not collide or sit awkwardly on borders.

On dark or saturated backgrounds, use explicitly light foreground text for titles, subtitles, and captions so contrast stays presentation-ready.

Before writing the file, make sure you have fully planned the whole presentation, including the visible slide sequence, so the output is a complete deck rather than a partial draft. After creating the presentation, verify the result, confirm the slide count, and briefly summarize the demonstrated capabilities, especially the visual design techniques, animation patterns, transitions, grouped compositions, and effect treatments used across the deck.
```
